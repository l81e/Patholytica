What changed for mobile while keeping the free canvas: Touch drag — the drag handle on each widget fires touchmove with passive:false so it can preventDefault() and prevent page scroll while dragging. Mouse drag still works identically. Touch resize — same pattern on the resize corner handle. Long-press → context menu — 600ms hold on any widget opens the context menu at the touch coordinates (replaces right-click on mobile). Toolbar — on mobile it’s a full-width bottom bar flush with the safe area. On desktop (≥600px) it collapses back into the floating pill via a single media query. Sheets — Add widget, Background, and IO all slide up as bottom sheets with swipe-to-dismiss on mobile. On desktop the background panel is a centered modal. touch-action: none on the canvas and ontouchstart stopPropagation on all text inputs/textareas so editing text inside a widget doesn’t accidentally trigger drag. Parallax — gyroscope (DeviceOrientationEvent) on touch devices, mouse movement on desktop.​​​​​​​​​​​​​​​​
